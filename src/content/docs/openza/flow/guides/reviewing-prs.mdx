Flow helps answer one question: which pull requests need my attention?

## Main views

| View | Purpose |
|------|---------|
| **Review Requests** | Pull requests where GitHub has requested your review |
| **Created** | Open pull requests created by you |
| **Recently Reviewed** | Pull requests you recently reviewed |
| **Recently Created** | Recent pull requests created by you |

## A simple workflow

<Steps>
1. Start with **Review Requests** to see what is waiting for you.
2. Open **Created** to check whether your pull requests need a response.
3. Search when you know the repository, author, or title.
4. Choose an organization when you want to narrow the list.
5. Open the pull request on GitHub when you are ready to review or make changes.
</Steps>

<Aside type="note">
Flow does not approve, comment, request changes, merge, or edit pull requests. Those actions remain on GitHub.
</Aside>

## Refresh

Select **Refresh** for an immediate update. If background mode is enabled, Flow can keep checking and notify you when a new review request appears.
