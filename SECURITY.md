# Security Policy

Please do not open public GitHub issues for security reports.

If you believe you have found a security issue affecting this website or the published Openza documentation, report it privately by emailing Deependra Solanky at the contact address listed on [solanky.dev](https://solanky.dev).

Include:

- a concise description of the issue;
- steps to reproduce;
- affected URLs, files, or commits;
- any relevant logs or screenshots.

This website repository does not accept general feature requests or content contributions. For product-specific security issues, use the security reporting process in the relevant Openza product repository.
